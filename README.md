# dots

React + TypeScript + Vite 기반의 프로젝트입니다.

## 시작하기

### 설치

```bash
npm install
```

### 개발 서버 실행

```bash
npm run dev
```

### 빌드

```bash
npm run build
```

### 미리보기

```bash
npm run preview
```

## 사용된 기술 스택

- **React** - UI 라이브러리
- **TypeScript** - 타입 안정성
- **Vite** - 빌드 도구
- **Tailwind CSS** - 스타일링
- **Storybook** - 문서화
- **Vitest** - 테스트
- **Prettier** - 코드 포맷팅
- **ESLint** - 코드 린팅

## 고려한 점

### 컴포넌트 위치, 구조

디자인 시스템이라고 생각하고, src/shared/ui 폴더 내에 위치하였습니다.
model폴더는 데이터와 타입 폴더를 관리하고, lib 폴더는 유틸함수를 관리합니다.

### Select 컴포넌트

Select 컴포넌트는 헤드리스 컴포넌트로 구현하여 유연성을 높였습니다.
디자인시스템이 정해지지 않아 스타일은 상위에서 진행하였습니다.
디자인 시스템이 정의되면 select 컴포넌트 내부에 스타일을 진행합니다.

#### 상태 관리 방식

value와 onChange는 외부에서 주입을 받고 컨텍스트 API를 통해 격리된 전역 상태로 관리합니다.  
value를 주입 받도록 한 이유는 select 내부에 상태를 가지게 될 경우 내부 value와 외부 value의 싱크를 맞춰야합니다.
하지만 외부에서 주입하면 싱크를 맞춰야하는 로직 등이 불필요해진다고 생각하여 주입받도록 했습니다.
부모컴포넌트에서 정보를 필요로 하지 않는 상태는 내부에서 격리된 전역 상태로 관리합니다.

### Modal 컴포넌트

Modal 컴포넌트 역시 헤드리스 컴포넌트로 구현하여 유연성을 높였습니다.

#### 상태 관리 방식

모달 구현에 트리거가 없어서 isOpen과 onClose는 외부에서 주입을 받아서 관리 합니다.
